说在开头的话

上一章都是虾扯淡，这一章开始上真格的了，当然我会尽量把生涩难懂的知识讲的简单有趣的

Hello World

不说别的，这是程序猿的信仰，咱们从它开始

$ #include<stdio.h>
  int main()
  {
     int num;
     num=0;
     printf("Hello World"); //打印 Hello World
     return 0;
  }

咱们从头开始

1.首先当然是 #includ<stdio.h> 了这个到底是什么意思呢？且听我细细道来：
假如你有个哆啦A梦的异次元口袋（想的美），这个口号的尽头是好多仓库A、B、C、D···，假如你这次想用A仓库里的道具，你需要先告诉它呀，你给它说：“嗨宝贝，能把你的异次元通道通往A仓库吗？”口袋很听话把通道通往A。然后你就可以随意从A里挑选好玩的东西了，来吧，给我一个使命必达盒子。。。

这个#include叫预处理命令，也就是你的计算机最开始会先处理这条命令，而后面的stdio.h相当于A仓库，而这句话的意思就是，计算机会先开始读这行代码，读完后，它会把已经寄存到内存里的stdio.h（头文件）这里面的东西（函数）都搬来，到时候你需要里面的什么东西（函数）都可以直接调用，然后你下面若是出现c语言的标准函数时（printf、scanf···），这个程序就可以运行了。

假如若没有这句话，你的异次元口袋搞不懂你想要什么，当然也就无法运行了。

（在普及一点课外知识，计算机这个学科里有这么一句话：避免重复造轮子。也可以说成：站在巨人的肩膀上。意思就是要善于使用前人以经建立好多，因为它已经经过了时间的考验，和众多前辈的考验，比方说c语言的这些函数，等你学习了指针之后，你也可以随意创造函数，但这些你自己创造的函数是经不过时间的考验的，不一定什么时候就会出现bug，然后你的程序就会崩溃，下一个崩溃的就是你了，在举个🌰吧，Adobe全家桶，假如今年2019年，你会用什么版本的呢？大部分外行人会选择本年头的也就是2019年的，而真正有经验的大虾则是会选择2017甚至2016版本的，因为这些老版本的兼容性好，很少会出现bug问题，而那些新版本的就不行了，你知道最痛苦的是什么吗？大半夜加班做的素材或设计的图案因为兼容性问题奔溃了。。。什么？你问为啥不说2020版的？2020版的ps搭载了机械学习，什么真一件抠图什么，嗯。。真香，妈妈再也不用担心我的学习了）

2.扯了半天别的，我们现在来说说int main（）这是什么吧

这个在书中给了一个很忽冷的介绍：就目前而言只需记住这是c程序的基本模块。也就是目前的话，你要写一个c的程序就要先写这个。当然我怎么可以告诉你个这个就结束了呢。我从网上找到了个比较不错的解释：“在C语言当中，一个程序，无论复杂或简单，总体上都是一个“函数”；这个函数就称为“main() 函数”，也就是“主函数”。比如有个“做菜”程序，那么“做菜”这个过程就是“主函数”。在主函数中，根据情况，你可能还需要调用“买菜，切菜，炒菜”等子函数。”

那么你会问int 是干嘛的呢？我这里先告诉你这是一个定义整形变量的数据类型，也许你有些懵了，我在解释一下看到程序后面有个return 0 了吗？这表示最后这个程序最后会以0作为结尾，而0又是一个整数，还记得int是整形变量的数据类型吗？这形成了一个循环。（多哲学呀，缘起即缘终，最开始就已经注定的结尾,各位请把哲学打在公屏上）
包含这个几个部分的才叫一个完整的程序

int main()
{

    return 0;
}

3.第三部分我要讲一讲c语言注释的写法

什么是注释？其实就是//，你要说它有什么用，其实还真没有什么用，但又确实有比较大的用处，嗯，可以说注释是一项非常有用的辅助工具，它存在的意义就是让你明白，你写了些什么，因为 // 后面的东西是不被c语言运行的，所以你可以在后面写任何东西，一个程序的代码行数是非常多的，无法避免发生，写到后面忘掉了变量之类的事情，或者说，回过头你再来看这段代码时，你都会说，这 ** 写的是什么，我怎么一点都看不懂它的意思，所以说会写注释也是一个很好的技能，将来工作时，你的项目经理爸爸也会要求你写代码注释的。这也是为了后期的运维着想。

注释一般有两种写法

一个是直接在程序后面加 //

/*
  另一种是像这样框起来
*/

当然注释在你找bug或者调试程序的时候也有奇效

4.花括号

也就是int mian（）下面的{ }，c语言规定所有c函数都要使用{ }来标记函数体的开始和结束，不能省略，而且还不能用其他的括号。

5.声明（起名字）

这个就比较重要了，所以要仔细👀呦！

有这样一个班级叫计算机，班里有个老师叫程序猿，一天程序猿看见了三个人走在一起，程序猿感觉他们很不错，就上去问：”哎，你们都叫什么呀？“为首的是个小胖子，一看是程序猿，就毕恭毕敬的回答：”我叫num“，程序猿一听感觉这个小胖子不错，就问他：”欧，那你想不想加入我们班呀？“小胖子一听：”那感情好啊，当然呀。”程序猿又说道：“你先别着急，你加入我们班有个条件，就是你要有个外号，叫小胖子，到时候你出现在班里的时候，别人一听是num就都知道你是个小胖子，班里还有和你外号一样的人，你们有一个共同的特性就是胖，所以如果你同意这样的要求我就把你加进去。“小胖子一听，说：”那好吧我同意。“

这时候小胖子旁边的两个人嚷嚷起来，原来是他们都想要加入程序猿的班级。程序猿就说：”你们要参加也可以，先把你们的名字告诉我。”这两个人呀一个叫2ca另一个叫for。程序猿一听说这不行，你们的名字一个太奇葩来了，另一个我们班里早有一个叫for的了，如果你要来就重名了。
两人一听，合计着不能加入，这不行呀，然后他们就把名字改了，改成_ca和For了，最后他们都顺利的加入到了程序猿的班中去了。

这个故事虽然编的很傻，但我把声明的规则都写在了里面。咱们先说简单点的

变量（num）

这件事从数学角度比较容易说清楚，函数，它有自变量和因变量，简单来说y=2x，x就是它的自变量，假如y=10的话，x=5，这连小学生都会算，那咱们换一个y=10a，还是当y=10的时候，a=？，你肯定会说a当然也等于5吗，这有什么区别吗？那么我问你a和x有什么不同？

这就是变量，和它叫什么没有关系，但你可以令x=5，也可以令x=7。然后这就有涉及到另外一个问题—赋值。

赋值（num=0）

赋值是c语言里的基操之一，意思就是“把值0赋给num”当然看起来很简单，那让我们看下面的代码

a=0;
b=1;
a=b;
b=a;

请问b=?呢,答案是1，当然这个答案并不重要，重要的是我们要从更深刻的角度去理解。

当你用int去定义num时,也就定义了num的空间大小，这里也许你会有疑问，这和空间大小有什么关系呢？int又是什么呢？当然这里你可以先看看，看不懂也没太大关系，后面会重点讲解的。

int代表32个格子（int有4字节大小，每个字节有8位）的仓库，仓库的名字叫num，每个格子可以放0或者1，比如你现在想把7（十进制）放进去，你放进去就给这个仓库赋值，7的二进制是0111，然后当你对他再次赋值的时候，之前放在仓库里的就清除了，而新的值进入到仓库中，就相当于又对这个仓库赋了一次值。

这时候聪明的你也许会问，会不会仓库装不下呀？

别着急，咱们下次再说。

命名规则（num）

命名长度：在c99的规定中，一个名字的长度只能小于63个字符，因为编译器只能识别前63个字符（应该没有程序猿会用这么长的命名吧，能写出这样程序的估计都收到了亿点打击）

命名名称：a.c语言不可以重复命名 b.c语言不允许使用库里面的函数，因为会导致冲突（如for···，但你可以使用For来命名，因为c语言名称区分大小写） c.允许使用小写字母、大写字母、数字、下划线（ _ ）
d.名称的第一个字母必须是字符或者下划线，不能是数字

数据类型（int）

嗯，怎么说呢？在数学上有实数、虚数或者整数小数计算机中有没有实数和虚数的区分我不知道，但是计算机在早年是无法整数和小数在一起表示的，这时候你可能会有许多❓，但是你要知道计算机的运算方法是0和1，也就是二进制，所以很多用人的思维很容易理解的事情对计算机来说是十分困难的，这也是自然语言是编程语言的顶峰一样。当然如果你想要更好的去理解整数和小数在计算机中的表示方法的话，可以去学习一下计算机原理这本书，这也是大学计算机考研的必修课，相信会对你有很大的帮助的。

继续回到咱们的数据类型上来，咱们先看这样一段代码

{

  int num;

  float num1;

}

你也许不知道什么意思，我在引用一下上面的故事内容

{
  小胖子 num;
}

你是否感觉到有一丝丝清楚了呢？

这个小胖子就相当于num的一个属性，咱们在用数学的角度去理解一下，当这个int 在 num 前面时就相当于告诉计算机说，num是一个整数，你给我记好了（int是整数型的数据类型），然后后面出现的num就只能是整数（先这么理解，后面可以推翻），咱们在看看num1，他前面的float是什么呢？上面说了计算机需要用两种方式来表示整数和小数，刚才我讲了int是表示整数的，现在我告诉你这个float其实表示的是小数，也就是计算机你给我听好了，下面的num1只能是小数欧。当然还有double（表示更精细的小数）char（表示字母）···

现在你是不是更加清晰一些了呢？当然加入你还是无法理解，这不是日子还长吗？你可以在今后的码代码中一遍一遍去体会，当然我总是说码代码，这里我不是说让你自己硬想，去闭门造车，而是要多借鉴，也就是多抄，当你抄的多了或者说见的多了，你会发现自己对编程这件事的理解程度也就加深了。
